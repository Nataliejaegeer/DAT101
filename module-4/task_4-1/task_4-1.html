1. Why is object-oriented development so important in programming?

    2. Object-oriented programming is important because it helps organize code in a better way. Instead of writing everything in one place, we can group related data and functions into classes. This makes the code easier to understand, reuse, and maintain. It is also useful for larger programs, because it reduces repetition and makes debugging easier.

  2. In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.

    3. The AccountType object is used to store different account types like normal, saving, and pension. It works like a collection of constant values. This is helpful because we avoid hardcoding text multiple times in the program. It also makes the code cleaner and reduces mistakes, since we always refer to the same values.

  3. Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.

    4. Encapsulation means hiding parts of the code so they cannot be changed directly from outside the class. This is important because it protects important data like the account balance. We only allow access through methods like deposit and withdraw. In JavaScript, we can create private variables by using the # symbol. Public methods are the ones we can call from outside the class.

  4. Explain what happens when we use the "new" command in JavaScript.

    5. When we use the "new" keyword, we create a new instance of a class. This means we get a new object with its own values and methods. For example, when we create a new bank account, it has its own balance and type. Without "new", the class would not work as intended.
